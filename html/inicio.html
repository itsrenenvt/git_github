PROGRAMACIÓN WEB

¿Qué es GIT y GitHub?



GIT

Es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando éstas tienen un gran número de archivos de código fuente.
Su propósito es llevar registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos.

GitHub

Es una plataforma de desarrollo colaborativo de software para alojar proyectos utilizando el sistema de control de versiones Git.
El código se almacena de forma pública, aunque también se puede hacer de forma privada, creando una cuenta de pago.

GIT CONFIG

Git trae una herramienta llamada git config que te permite obtener y establecer variables de configuración, que controlan el aspecto y funcionamiento de Git. Estas variables pueden almacenarse en tres sitios distintos:

  • Archivo /etc/gitconfig: Contiene valores para todos los usuarios del sistema y todos sus repositorios. Si pasas la opción --system a git config, lee y escribe específicamente en este archivo.

  • Archivo ~/.gitconfig file: Específico a tu usuario. Puedes hacer que Git lea y escriba específicamente en este archivo pasando la opción --global.

  • Archivo config en el directorio de Git (es decir, .git/config) del repositorio que estés utilizando actualmente: Específico a ese repositorio. Cada nivel sobrescribe los valores del nivel anterior, por lo que los valores de .git/config tienen preferencia sobre los de /etc/gitconfig.

Esto es importante porque las confirmaciones de cambios (commits) en Git usan esta información, y es introducida de manera inmutable en los commits que envías.


Identidad

Lo primero que se debe hacer cuando se instala Git es establecer nuestro nombre de usuario por medio del comando:

git config --global user.name "nombreusuario"


E-mail

Otro parametro importante que se debe configurar en Git es nuestra dirección de correo electrónico.:

git config --global user.email email@email.com


Editor

Ahora que tu identidad está configurada, puedes elegir el editor de texto por defecto que se utilizará cuando Git necesite que introduzcas un mensaje. Si no indicas nada, Git usa el editor por defecto de tu sistema.

git config --global core.editor editor


Comprobando cambios

Si quieres comprobar tu configuración, puedes usar el siguiente comando para listar todas las propiedades que Git ha configurado:

git config --list

GIT INIT

Puedes obtener un proyecto Git de dos maneras.

  • La primera: Tomar un proyecto o directorio existente e incializarlo como un repositorio de Git.

  • La segunda: Clonar un repositorio Git existente desde otro servidor (GitHub).

Inicializar un repositorio.

Si estás empezando el seguimiento en Git de un proyecto existente, necesitas ir al directorio del proyecto y escribir:

git init [url]

Esto crea un nuevo subdirectorio llamado .git que contiene todos los archivos necesarios del repositorio un esqueleto de un repositorio Git.Todavía no hay nada en tu proyecto que esté bajo seguimiento.

GIT CLONE

Clonando un repositorio existente.

Si deseas obtener una copia de un repositorio Git existente por ejemplo, un proyecto en el que te gustaría contribuir el comando que necesitas es:

git clone [url]

Git recibe una copia de casi todos los datos que tiene el servidor. Cada versión de cada archivo de la historia del proyecto es descargado cuando ejecutas git clone. En su interior, descarga toda la información del repositorio clonado, y saca una copia de trabajo de la última versión. Si te metes en el directorio creado, verás que están los archivos del proyecto, listos para ser utilizados. Si quieres clonar el repositorio a un directorio con otro nombre que no sea el que tiene por defecto, puedes especificarlo con la siguiente opción de línea de comandos:

git clone [url] minombre

GIT BRANCH

¿Qué es una rama?

Una rama Git es simplemente un apuntador móvil apuntando a una confirmacion (commit). La rama por defecto de Git es la rama master. Con la primera confirmación de cambios que realicemos, se creará esta rama principal master apuntando a dicha confirmación. En cada confirmación de cambios que realicemos, la rama irá avanzando automáticamente. Y la rama master apuntará siempre a la última confirmación realizada.


¿Cómo crear una nueva rama?

Supongamos que estamos trabajando en un proyecto y queremos añadir nueva funcionalidad al mismo. La forma adecuada de hacerlo con Git es crear una nueva rama con el nombre de la nueva funcionalidad donde añadiremos nuestros cambios y después cambiaremos a ella. Al crear una nueva rama simplemente se crea un nuevo apuntador para que lo puedas mover libremente. Por ejemplo, si quieres crear una nueva rama. Usarás el comando:

git branch [nombre]

¿Cómo cambiar entre ramas?

Para saltar de una rama a otra, tienes que utilizar el comando:

git checkout [nombrerama]

¿Cómo ver todas las ramas?

Para ver todas las ramas existentes dentro de nuestro repositorio basta con ejecutar el siguiente comando:

git branch -a

¿Cómo eliminar una rama?

Cuando mezclamos las ramas con las que estamos trabajando, y en caso de que no haya conflictos, podemos eliminar la rama que hayamos creado para realizar nuestros cambios por que no vamos a necesitarla más, con el siguiente comando:

git branch -d [nombrerama]

Atajos

Existe un atajo para crear una rama que aún no existe y posicionarnos en ella rapidamente usando directamente el comando checkout que consiste en pasar el parámetro -b en la llamada.

git checkout -b [nombrerama]

GIT STATUS

Comprobar el estado de los archivos.

La herramienta para determinar qué archivos están y en qué estado es el comando:

git status

Si ejecutas este comando justo después de clonar un repositorio, deberías ver algo así:

on branch [rama]
nothing to commit, working directory clean

Esto significa que tienes un directorio de trabajo limpio, en otras palabras, no tienes archivos bajo seguimiento y modificados. Git tampoco ve ningún archivo que no esté bajo seguimiento, o estaría listado ahí. Por último, el comando te dice en qué rama estás.

GIT ADD

Clonando un repositorio existente.

Para empezar el seguimiento o añadir un nuevo archivo o los cambios generados dentro de este, se usa el comando:

git add [nombrearchivo]

Si queremos agregar todos los cambios generados en solo una petición, se pueden utilizar los siguientes comandos:

git add --all
git add --a

Si se vuelve a ejecutar el comando git status, verás que tus archivos estan ahora bajo seguimiento y preparados.

GIT COMMIT

Confirmando cambios.

Para confirmar o guardar los cambios hechos en nuestro repositorio se utiliza el comando:

git commit -m [mensaje]

El parameto -m indica que con el commit se envia un mensaje, es recomendable agregar uno, para que otro participante en el proyecto sepa de antemano que cambios fuerón generados. Si ejecutamos git status podremos observar que los cambios han sido guardados.

GIT PUSH

Trabajando con repositorios remotos.

Para poder colaborar en cualquier proyecto Git, necesitas saber cómo gestionar tus repositorios remotos. Los repositorios remotos son versiones de tu proyecto que se encuentran alojados en Internet o en algún punto de la red. Puedes tener varios, cada uno de los cuales puede ser de sólo lectura, o de lectura/escritura, según los permisos que tengas. Colaborar con otros implica gestionar estos repositorios remotos, y mandar (push) y recibir (pull) datos de ellos cuando necesites compartir cosas.

Añadir repositorios remotos.

Si quieres añadir tu repositorio local a un repositorio remoto alojado en los servidores de GitHub ejecuta el siguiente comando:

git remote add origin [urlservidor]

Para añadir un nuevo repositorio Git remoto, asignándole un nombre con el que referenciarlo fácilmente, ejecuta:

git remote add origin [nombre] [urlservidor]

Enviando al repositorio remoto.

Cuando tu proyecto se encuentra en un estado que quieres compartir, tienes que enviarlo a un repositorio remoto. El comando que te permite hacer esto es sencillo:

git push [nombre-remoto][nombre-rama]

Si quieres enviar a cualquier rama pero a tu servidor origen (origin), ejecutarías esto para enviar tu trabajo al servidor:

git push origin [rama]

GIT FETCH

Actualizando repositorio local con remoto.

Para recuperar datos de tus repositorios remotos puedes ejecutar

git fetch

Este comando recupera todos los datos del proyecto remoto que no tengas todavía. Después de hacer esto, deberías tener referencias a todas las ramas del repositorio remoto, que puedes unir o inspeccionar en cualquier momento.

GIT PULL

Actualizar y fusionar los cambios remotos.

Para recuperar y unir automáticamente la rama remota con tu rama actual al commit m´s nuevo, utiliza el siguiente comando.

git pull

Recupera la información del servidor del que clonaste, y automáticamente se intenta unir con el código con el que estás trabajando actualmente.

GIT MERGE

Mezclando ramas.

Si la parte del trabajo o el trabajo como tal ya esta terminado y desear unir los cambios de una rama con otra puedes fusionar los cambios de una rama con otra. Para fusionar una rama o má con la rama que se tiene activa, se utiliza el comando:

git merge [rama]

Si los cambios de una rama con otra han sido fusionados, la o las ramas que ya no se vayan a ocupar pueden ser eliminadas.

Deshacer un Merge.

En ocasiones, queremos deshacer un merge por que nos equivocamos o otra gente de nuestro equipo de trabajo se equivoco y encontraremos muchos conflictos a la hora de intentar seguir mezclando ramas o de trabajar normalmente. En esas ocasiones querremos deshacer de forma total el problema. Git viene nos facilita el comando reset. Para hacer un reset de nuestro directorio de trabajo y volver atrás hasta antes de intentar mezclar las ramas, solamente debemos de ejecutar git reset.

git reset --hard HEAD

El parámetro --hardse asegura de que tanto nuestro índice de archivos como el directorio de trabajo cambien para que coincidan con lo que era antes del merge. Por defecto solo se restablece el índice, dejando los archivos parcialmente mezclados en el directorio de trabajo.
